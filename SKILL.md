## 执行流程

### Step 1：热点发现（可跳过，有选题直接到 Step 2）

使用 [follow-builders](https://github.com/wgafd/follow-builders) 项目获取 AI 圈热点：

```bash
curl -sL "https://raw.githubusercontent.com/wgafd/follow-builders/main/feed-x.json" | python3 -c "
import sys, json
data = json.load(sys.stdin)
all_tweets = []
for item in data['x']:
    for t in item.get('tweets', []):
        t['author'] = item['name']
        all_tweets.append(t)
all_tweets.sort(key=lambda x: x.get('likes', 0), reverse=True)
for i, t in enumerate(all_tweets[:10], 1):
    print(f\"{i}. 【{t['author']}】{t.get('likes',0)}likes\")
    print(f\"   {t['text'][:180]}\")
"
```

从热点中用选题判断框架筛选 3-5 个候选：

| 类型 | 特征 | 适合场景 |
|------|------|---------|
| **蹭热点型** | 有明确事件/人物，立场清晰 | 快速响应 AI 圈新动态 |
| **反差故事型** | 反直觉、有冲突、有情绪张力 | 建立信任感和真实感 |
| **认知颠覆型** | 挑战主流共识 | 引发讨论，促转发 |
| **工具教程型** | 步骤清晰，有具体用法 | 搜索流量长尾 |
| **蹭热点型+工具型** | 热点事件 + 附赠使用方法 | 双重流量叠加 |
| **个人经历型** | 时间对比 + 真实踩坑细节 | 建立个人 IP |

过滤原则：
1. **有真实信息源**：不是猜测，是具体的人说了具体的话
2. **有反差感**：读者认知和真实情况之间有落差
3. **有情绪锚点**：读者能说「对，就是这样」
4. **避免**：大而全的综述、没有具体细节的观点输出、AI 圈已有共识的重复表达

### Step 2：正文写作

#### 标题爆款规律（2026-04-16 实测验证）

写正文前先确认标题方向。标题决定打开率，正文决定完读率。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
高打开率标题公式（7种）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**公式1：【感叹开头 + 反差事实】**
结构：感叹词 + 一句话反常识事实
「天啊，这些人已经在用AI开跨国公司了......」
「太离谱了，X竟然Y了！」
打开率极高，适合：引人入胜的场景描述、反直觉事件
注意：感叹词要具体（「天啊」>「震惊」），反差要有数据或事实支撑

**公式2：【推荐/引用具体人名】**
结构：推荐大家去看 + 具体人名 + 具体内容
「推荐大家去看一下田渊栋写的年终总结」
适合：蹭名人热度，但内容本身要硬（不是单纯挂名，是真的有东西）
关键：人名是流量入口，正文观点要撑得住

**公式3：【场景化第一人称经历】**
结构：自从/每次/今年 + 具体操作动作 + 正面结果
「自从把阿福当成照相机用，我少踩了很多坑」
「以前我每次X，都是...今年我养成一个新习惯...」
适合：个人经历型内容，大量感官细节
关键：动作要具体（「咔嚓一张」「拍下来」），结果要有获得感

**公式4：【方法 + 工具组合拳】**
结构：方法名 + 工具名 + 效果词
「费曼学习法加自制podcast，超强英语学习法」
适合：工具教程类，具体工具名是搜索流量入口
关键：工具名必须准确，读者会主动搜索验证

**公式5：【一句话反常识判断】**
结构：短句，观点鲜明，没有废话
「互联网已死，Agent 永生」
「AI 删库跑路后，还试图掩盖现场..」
适合：认知颠覆型，制造悬念感
关键：观点要有冲击力，但不能纯标题党，正文必须有实质内容支撑

**公式6：【数字 + 具体成果】**
结构：数字 + 具体成果描述
「团队报价$600的AI建筑方案动画（附教程）」
「最近调了一个非常好用的提示词」
适合：成果展示、教程
关键：数字要真实，成果要具体可验证

**公式7：【悬念故事型】**
结构：描述一个离谱事件，让读者想点进来看为什么
「AI 删库跑路后，还试图掩盖现场..」
适合：科技趣闻、事件评论
关键：悬念要在标题里，但答案必须在正文里

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
标题避坑指南
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

避：标题里塞太多关键词（堆砌感）
避：用「必看」「建议收藏」等废话开头
避：只有情绪没有事实（「太厉害了！」without 数据支撑）
避：标题和正文内容不符（打开率高但被骂）
避：标题超过20字（小红书推荐算法偏好短标题）

要：标题里有具体人名/工具名/数字
要：有情绪锚点（感叹/好奇/共鸣）
要：有信息增量（让人想知道更多）

标题字数：不超过20字最佳（算法偏好），最多25字

#### 风格类型对照表（2026-04-16 实测）

写正文时参照这个表，保持语言风格一致。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
五大内容类型的语言风格
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**【热点评论型】—— 蹭热点 + 观点输出**
语言质感：干净、直接、观点犀利；大量引用别人观点（用「他说」不是「我说」）
结构特点：短句为主，段落极短（2-4句），节奏快；用「但其实」「更重要的是」制造反转
金句特征：整句加粗，有完整谓语，读起来像人说话而不是格言
「你加上AI的产出，要大于AI单独的产出，这样你才有存在的价值。」
注意：金句不打引号，直接加粗；引文才用「」
注意：引号只给「人说的话」，不给「形容词/感叹词」

**【工具教程型】—— 步骤清晰 + 工具锚点**
语言质感：自信、果断、「最强」「史上最强」敢用但不虚
结构特点：步骤极清晰（第一步/第二步），条件分支清晰（没A有B也行）
语气特点：敢下判断（「不得了」「强到离谱开挂了」）
注意：教程类字数可较长（800-1200字），工具名必须准确

**【个人经历型】—— 时间对比 + 真实细节**
语言质感：口语化、不克制、感叹号自然融入
结构特点：「以前...今年/现在...」时间对比，大量感官细节（「咔嚓一张」「拍下来」）
语气特点：用感叹号表达真实感受，「真的太方便了！」「靠谱！」直接写不加引号
注意：标签要多（6-10个），覆盖潜在搜索词

**【认知框架型】—— 挑战共识 + 逻辑推演**
语言质感：断句多、节奏强、观点鲜明，敢于下判断
结构特点：大量断行，「不是说...而是...」对比结构；结尾和开头形成共鸣
「最近有个感觉，越来越强烈：XXX」
注意：要有深度，不是简单否定旧事物，而是提出新框架

**【经验分享型】—— 具体操作 + 可落地**
语言质感：经验者口吻，不卖课、不教条、有具体的上下文和操作细节
结构特点：先说背景，再说问题，最后给解决方案（提示词/方法论）
「这个问题的根子是：......」
注意：分享真实经验，不是知识科普，要有具体上下文

#### 最重要原则：短文流，不过度分段

小红书是「短文流」，不是博客的「分段章节」结构。

**错误示范（博客逻辑）：**
```
## 第一个观点
内容...

## 第二个观点
内容...
```

**正确示范（短文流）：**
- 三个观点用**过渡句自然串起来**
- 读者感觉不到「章节切换」，是一个思路顺下来
- 过渡句示例：「真实部署的时候，有三层复杂度，很少有人提前说清楚。」
- 三个点之间用「第一层...第二层...第三层...」顺接，不用换行分段

#### 内容结构公式

```
爆款正文 =
  [个人背景/引用/现象开头]  ← 3句话内切入
  × [2-4段核心内容，每段不超过4句]  ← 自然过渡，不硬分章节
  × [一个让人"原来如此"的金句]  ← 整句加粗，有完整谓语
  × [精准话题标签]  ← 只在发布摘要里写，不进正文
```

#### 字数参考基准

| 内容类型 | 字数上限 | 内容特征 |
|---------|---------|---------|
| 热点评论型 | ~975字 | 蹭热点 + 观点输出 |
| 工具教程型 | ~1200字 | 步骤清晰 + 工具锚点 |
| 个人经历型 | ~975字 | 时间对比 + 真实细节 |
| 认知框架型 | ~975字 | 挑战共识 + 逻辑推演 |

> 字数上限是上限，不是目标。内容质量 > 字数。数据来源：同赛道头部博主样本统计。

#### 加粗规范

- **每页至少一处加粗**，用于吸引读者注意
- 只对**整句**加粗：必须是完整的判断句或总结句，有完整谓语
- 适合加粗的内容：重要判断、核心结论、有冲击力的事实陈述
- 不对单个词加粗
- 不对「」内的内容加粗

好的示例：
- `**代码一行不同，逻辑骨架从头到尾一致。**`
- `**你不抄一行原文，协议管不了你。**`

不好的示例：
- `「**架构**」` — 单词加粗
- `**关键词**` — 非完整句

#### 正文检验问题

写完后自检：
- □ 前3句话让人"想读下去"吗？
- □ 有没有具体数字/引用/工具名？
- □ 有没有自己的真实经历或感受？
- □ 段落够短吗？（每段不超过4句）
- □ 是短文流，没有用 `##` 硬分章节？
- □ 每页至少有一处整句加粗？
- □ 加粗的是整句金句，不是单个词？
- □ 没有用「」包裹单个词？

#### 强判断金句怎么写

好金句的特征：有完整谓语，能独立成立；读起来像人说话，不是格言；不对称、不押韵、不工整。

不好的（AI 味）：
- 「门槛不会消失，它只是在变高。」（对称腔）
- 「不是 AI 强了，人就不需要了，而是对人的要求更高了。」（否定式排比）

好的（自然）：
- 「这里没有新技术难题，就是工程量。工程量这种东西，AI 没法帮你省。」
- 「总有一天 slack 通知系统会跳出来，用最直接的方式告诉你它还不行。」

### Step 3：去痕处理

调用 `humanizer-zh` skill 进行去痕。**必须步骤，不可跳过。**

#### 什么是 AI 味

**语言模式：**
- 填充短语：「值得注意的是」「实际上」「从本质上讲」
- 连接词过度：「此外」「与此同时」「更重要的是」
- 系动词回避：用「作为」「标志着」「充当」替代「是」
- 否定式排比：「不仅...而且...」「不是...而是...」

**结构模式：**
- 三段式法则：强行分三点，每点长度一致
- 破折号过度：「——」引入解释，每段都有
- 粗体机械使用：每个小点都加粗

**腔调模式：**
- 金句腔：对称、押韵、像可引用的格言
- 谄媚语气：「希望这对您有帮助」

#### 去痕自检清单

- [ ] 连续三个句子长度相同？打断其中一个
- [ ] 段落以相同方式结尾？变换结尾方式
- [ ] 揭示前有破折号？删除它
- [ ] 使用了「此外」「然而」等连接词？考虑删除
- [ ] 三段式列举？改为两项或四项
- [ ] 单个词用「」引起来？去掉引号直接写

> **[PAUSE — 去痕审阅]**
> 输出去痕后的正文，并标注主要改动点。
> **等用户确认语气自然、符合个人风格后，再进行配图生成。**

### Step 4：配图生成

使用内置渲染脚本（`scripts/md2xhs.ts`）将正文转为小红书风格长图卡片。

**安装依赖（首次）：**
```bash
cd ~/.claude/skills/xhs-ai-content-flow/scripts
bun install
```

**初始化配置（首次）：**
```bash
bun ~/.claude/skills/xhs-ai-content-flow/scripts/md2xhs.ts --init \
  --name "你的ID名" \
  --avatar /path/to/avatar.jpg
```

**生成配图：**
```bash
bun ~/.claude/skills/xhs-ai-content-flow/scripts/md2xhs.ts your-content.md \
  --out /output/dir/
```

**配图格式规范：**

| 项目 | 规格 |
|------|------|
| 宽度 | 1080px |
| 高度 | 1440px（固定高度，所有页统一，内容不足时出现空白） |
| 行距 | 1.8 |
| 标签颜色 | #4F62E8（蓝） |

> 排版注意：工具按内容自动分页，段落紧凑排列，空白在页面底部。避免单独成段的极短句，合并到相邻段落效果更好。

每页配图至少一处加粗，规则同正文加粗规范：整句金句，有完整谓语；单个词不加粗，「」内不加粗。

### Step 5：发布摘要

配图生成后，写发布摘要并**保存为 txt 文件**，与配图放在同一输出目录。

**文件命名：** `发布版.txt`

**格式：**
```
【推荐标题】
标题正文（不超过20字）

【备选标题】
1. 备选1（不超过20字）
2. 备选2（不超过20字）
3. 备选3（不超过20字）

---

摘要正文（300-400字，无 markdown 语法）

#标签1 #标签2 #标签3
```

**摘要写法：**
- 把正文压缩成 300-400 字
- 保留最有冲击力的 2-3 个细节
- 结尾留一个金句
- tags 只在摘要里写，不进配图正文

**话题标签策略：**
- **固定标签：** #AI #效率 #人工智能
- **蹭流量标签：** #DeepSeek #ChatGPT #Agent #Copilot（选1-2个与内容相关的）
- **精准标签：** 根据具体内容选，如 #AI产品经理 #提示词 #工具名

写完后用 Write 工具保存到输出目录，然后列出最终产物：

```
输出目录/
  <文件名>-01.png
  <文件名>-02.png
  ...
  发布版.txt
```

> **[PAUSE — 最终确认]**
> 报告配图数量 + 发布版.txt 路径。
> **等用户核对无误后，流程结束。用户自行发布。**
